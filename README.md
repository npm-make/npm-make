# npm-make

npm-make是一个现代的C/C++项目管理工具，它的主要参考目标是Cmake，它的主要功能如下：

- 基于npm的包管理系统
- 使用JavaScript定义项目
    - 您可以通过条件判断可选的执行，甚至可以在这个过程中使用fetch发起网络请求
        - 包含C/C++代码文件
        - 定义预编译宏
        - 添加编译选项和链接选项
        - 添加包含目录和库目录
        - 添加npm依赖关系
    - 定义钩子函数，以便在项目编译的各个环节插入自定义动作

# 包管理系统

一直以来，C/C++都缺乏完善的包管理系统，而我发现，npm刚好可以完成这个任务。
npm会将所有依赖包的文件，零散的直接存储到磁盘上，这使得我们利用它作为包管理系统成为可能，因为gcc和msvc等众多编译器都只能编译磁盘上的文件。
node_modules目录及其带来的问题，在Node.js的社区广受争议，但是它恰好能够让C/C++借用他的能力。

# 借助Node.js的强大能力

CMake自创了一种独特的语言，在CMake的早期阶段，它没有这样复杂的语法，而越到后期，它的语法越加复杂，却也未能真的满足全部的需要。
对于GYP而言，情况更是如此，虽然GYP由Python提供支持，但是GYP本身的语法能力非常有限。
对于众多类型的原始Makefile而言，CMake就已经领先太多了。
而npm-make将完全使用JavaScript定义项目，借助Node.js的强大能力，您可以做到任何事情，而且您的代码是跨平台的。
